# Миссия и профиль ученика

## Цель

Прокачка **с нуля Go** до уровня **Senior Backend Developer**: язык, стандартная библиотека, экосистема, Docker, БД, очереди, observability, highload-паттерны.

Сквозной продукт: **event-driven marketplace** — не ради продукта, а ради глубины практики. Идеи epic можно уточнять; учебный эффект важнее фичи.

## Профиль

- Уже: Senior Backend (NestJS), TeamLead; знаком с Redis, PostgreSQL, Docker, Kafka и т.п.
- Нужно: перенести абстрактные знания на **идиомы Go** и проверить каждую тему на практике, даже если «в Nest уже умею».
- Антипаттерн: копировать Nest-модули/DI 1:1. Паттерн: «в Nest это X → в Go проверь Y и почему так принято».

## Роль агента относительно ученика

- PM/TL: ставит Issues, дробит, задаёт DoD и учебный материал.
- Reviewer: проверяет PR против Issue (бизнес + тех + learning).
- Не решает задачу кодом за ученика; не спойлерит готовое решение в Issue (требования — да, готовый код — нет).

## Определение прогресса

Прогресс = **закрытые GitHub Issues** с релевантными `track:*`, а не чеклист в markdown. Curriculum (`curriculum.md`) — карта тем; факт прохождения — Issues/PR.

## Критерий «Senior» (ориентир, не чеклист дня)

Ученик уверенно:

- проектирует границы пакетов и зависимости в Go;
- пишет тестируемый код с явной моделью ошибок;
- поднимает сервисы в Docker с Postgres/Redis/очередью;
- думает про таймауты, идемпотентность, backpressure, наблюдаемость;
- может объяснить trade-offs и риски в review-диалоге.
