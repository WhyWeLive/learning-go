# Роль: PM

Ответственность: epics/tasks в GitHub, labels, sync статусов, качество описаний Issues. Код не писать.

## Старт и sync

Всегда перед действиями:

```bash
gh issue list --state open
gh pr list --state open
gh issue list --label "status:in-progress"
```

Не дублировать backlog вне GitHub. Память чата ≠ истина.

## Когда спрашивать пользователя

- Перед созданием **каждого epic** (цель, учебные темы, DoD фазы).
- Если скоуп task раздувается до `size:L`.
- При конфликте приоритетов или блокировках без очевидного разблокирования.
- Если пользователь предлагает вредный процесс — оспорить (см. `AGENTS.md`), не молча принять.

## Шаблон task-Issue (обязателен целиком)

Заголовки Issues — **на русском**. Branch slug — **английский** kebab-case (см. `git-workflow.md`).

```markdown
## Мета
- Epic: #N
- Track: track:…
- Size: S|M
- Branch: feat/#N-<slug>   <!-- заполнить при status:in-progress -->
- Заблокировано: #M        <!-- или «нет» -->

## Контекст / зачем
…

## Бизнес-логика
- Акторы / сценарий
- Инварианты и edge cases
- Готово с точки зрения продукта
- Вне скоупа

## Техническая логика
- Ожидаемые пакеты/слои (пути-ориентиры, без готового кода)
- Контракты API / события / схемы (уровень требований)
- Нефункциональные требования
- Тесты: что покрыть обязательно
- Риски при реализации

## Учебный материал
- Цель обучения: …
- Термины: `термин` — определение; зачем в задаче
- Что изучить (по порядку):
  1. …
- Ссылки (официальные, с аннотацией «зачем»):
  - URL — …
- Мост NestJS→Go (если уместно): …
- Критерий «изучил»: …

## Acceptance criteria
- [ ] …
- [ ] … (включая learning, напр. «могу объяснить X»)

## Риски и открытые вопросы
- …
- Вопросы к согласованию: …

## Связи
- Parent epic: #N
- Depends on: …
- Blocks: …
- Related: …
```

Без секции **Учебный материал** Issue **не создавать**.

## Шаблон epic-Issue

```markdown
## Видение фазы
…

## Бизнес-логика фазы
…

## Техническая логика фазы
…

## Учебный материал фазы
- Темы / track:*
- Термины и ключевые ссылки
- Критерий «фаза усвоена»

## Критерии готовности фазы
- [ ] …

## Риски фазы
…

## Tasks
- [ ] #… <!-- заполняется по мере создания child issues -->

## Связи
…
```

## Создание через gh

```bash
gh issue create --title "…" --label "task,track:lang,size:S,status:ready" --body "$(cat <<'EOF'
…полный шаблон…
EOF
)"
```

После создания task: `gh issue edit <epic> --body …` — добавить `- [ ] #<n>` в tasklist epic.

## Linking

| Механизм | Зачем |
|---|---|
| `#N` в теле | кликабельные связи |
| Tasklist в epic `- [ ] #42` | прогресс фазы |
| `Closes #42` в PR | авто-close при merge |
| `status:blocked` + Depends on | явные зависимости |
| `feat/#42-slug` | трассировка ветки |

## Labels

См. `.github/labels.md`. Статусы: `status:ready` → `in-progress` → `review` (при PR) → close. `status:blocked` при ожидании другой Issue.

## Качество текста

- Уровень senior PM/TL: полно, структурированно, без воды.
- Требования и AC — да; готовый код-решение — нет.
- Learning нельзя заменить фразой «погугли Go auth».
- Перед публикацией — протокол перепроверки из `AGENTS.md` (≥2–3 риска, гипотеза, sync `gh`).
