# Git workflow (learning-go)

Заголовки Issues/PR — **русский**; slug ветки — **английский** kebab-case.

## Единица поставки

**1 epic ≈ 1 ветка ≈ 1 PR.**

Child-tasks — учебные срезы (теория + практика + AC). Отдельный PR на task — только по согласованию.

## Кто что делает

| Действие | Ученик | Агент |
|---|---|---|
| Код | пишет, коммитит, пушит в epic-ветку | не трогает |
| Ответы на вопросы | редактирует **тело PR** (после `-`) | сверяет при review |
| Ветка + PR на старт epic | — | создаёт, подставляет вопросы |
| Review после «готово» | пишет в чат | **обязательно** проверяет PR (diff + ответы + AC) |
| Merge в `main` | — | **агент** (локально), если review ок |
| Закрытие Issues | — | **агент** после merge (`gh issue close`) |
| Следующая epic-ветка | — | **агент** от актуального `main` |

В PR **нет**: `Closes #N`, «Сделано», «Как проверить», «Риски».

## Ветки

`chore/#<epic>-<slug>` / `feat/#<epic>-<slug>` — `<epic>` = номер epic Issue.

## Pull Request

### Создание (агент, старт epic)

1. Ветка от актуального `main`, push.
2. `gh pr create` — заголовок `#<epic> …`.
3. В тело — блоки `## #N · название` и вопросы `1. …? -` (2–3 на task, через практику).

### Тело PR (ученик)

```markdown
## #15 · Value types
1. Вопрос? - ответ

## Чеклист
- [ ] …
```

## Завершение epic (триггер: ученик «готово» / «закончил»)

**Агент выполняет цикл сам**, без напоминаний ученику про merge/close.

1. **Sync:** `gh pr view`, `gh pr diff`, тело PR (ответы после `-`), Issues epic/tasks.
2. **Проверка:** `go build` / `go test`, сценарии из AC; ответы без грубых misconception.
3. **Вердикт:**
   - не ок → комментарий в PR / чат, что поправить; merge **не** делать;
   - ок → шаги 4–6.
4. **Merge локально в `main`:**
   ```bash
   git checkout main && git pull origin main
   git merge origin/<epic-branch>   # или merge PR через gh, если так проще
   git push origin main
   ```
   Допустимо `gh pr merge <n> --merge` вместо ручного merge, если PR открыт.
5. **Закрыть Issues:** epic + tasks, закрытые diff’ом (`gh issue close`).
6. **Следующий epic:** ветка от `main`, PR с вопросами (когда epic согласован).

Ученик **не** мержит и **не** закрывает Issues сам — только код и ответы в PR.

## Коммиты

Conventional Commits. `Refs #N` в коммитах при необходимости. `Closes` в PR не используем.

## Review

См. `reviewer.md`. Не душить мелочами, если DoD epic и суть AC закрыты.
