# Git workflow (learning-go)

Заголовки Issues/PR — **русский**; slug ветки — **английский** kebab-case.

## Единица поставки

**По умолчанию: 1 epic ≈ 1 ветка ≈ 1 PR.**

- Child-tasks — учебные срезы внутри epic (теория + практика + AC).
- Отдельный PR на task — только по согласованию.

Цель — **учиться и получить review по сути**, не набирать ритуал merge’ей.

## Кто что делает

| Действие | Ученик | Агент |
|---|---|---|
| Код приложения | пишет, коммитит, пушит | не трогает |
| Ветка + PR на старт epic | — | создаёт (`gh`), подставляет контрольные вопросы в PR |
| `Closes #N` в PR | — | **не используем**; Issues закрывает агент после merge |
| «Как проверить» в PR | — | агент проверяет сам (`build`, сценарии из AC) |
| Чеклист в PR | отмечает самопроверку | перепроверяет независимо |
| «Учебный итог» | отвечает на вопросы в PR | сверяет с Theory Issues |
| Review / close Issues | merge по процессу | `gh pr review`, `gh issue close` после merge |

## Ветки

| Префикс | Когда |
|---|---|
| `feat/#<n>-<slug>` | продуктовая фича |
| `fix/#<n>-<slug>` | баг |
| `chore/#<n>-<slug>` | инфраструктура / каркас |
| `docs/#<n>-<slug>` | документация |
| `refactor/#<n>-<slug>` | рефакторинг |

- `<n>` — номер **epic**.
- Ветку и PR на старт epic **создаёт агент** от актуального `main`.
- Epic → `status:in-progress` при старте; PR открыт → `status:review`.

Пример: epic `#7` → `chore/#7-marketplace-skeleton`.

## Коммиты

Conventional Commits. В теле коммита: `Refs #N` при необходимости. **`Closes` — не в PR и не обязателен в коммитах** (закрытие Issues — агент).

## Pull Request

### Создание (агент)

При старте epic:

1. `git checkout -b chore/#<epic>-<slug>` от `main`, push.
2. `gh pr create` с заголовком `#<epic> …`.
3. В тело PR — **контрольные вопросы** из Theory всех tasks epic (блоками `#8`, `#9`, …). Ученик **не переписывает** вопросы — только ответы.

### Тело PR (ученик заполняет)

Короткие секции, без лишнего жирного. Вопросы — агент; ответы — под номером или с новой строки.

```markdown
Epic #7 · tasks #8–#11

## Сделано
…

## #8
1. Вопрос? -
2. Вопрос? -

## Чеклист
- [ ] …
```

Нет в PR: `Closes #N`, «Как проверить», «Риски» (риски — в review агента).

### После merge (агент)

1. Review уже был до merge.
2. Закрыть epic и tasks, реально закрытые diff’ом: `gh issue close …`.
3. Синхронизировать labels / checklist в epic.

## Review

См. `reviewer.md`. Агент сам прогоняет проверки из AC; не требует от ученика дублировать команды в PR.
